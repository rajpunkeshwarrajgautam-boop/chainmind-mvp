# Pilot SOW / Order Form language (template, non-legal)

Use with counsel. This is a starting point for paid pilot agreements.

## Positioning
- Service tier: **Paid pilot / early access for selected customers**.
- The service is **not** represented as enterprise-ready, certified, or under production SLA.

## Scope
- Included:
  - API access for forecasting workflows
  - basic support channel
  - agreed pilot use cases and user seats
- Excluded unless explicitly added:
  - SOC 2 / ISO certification commitments
  - custom ERP certification work
  - formal uptime SLA credits
  - custom security controls beyond documented MVP features

## Pilot limitations
- Features may change during pilot based on feedback.
- Some workflows are best-effort while product stabilizes.
- Availability target is operational best effort, not contractual SLA.

## Data and security
- DPA and subprocessor inventory attached/referenced:
  - `docs/legal/dpa-template.md`
  - `docs/legal/subprocessors.md`
- Customer remains responsible for data quality and lawful basis for data provided.

## Support and incident handling
- Primary support channel and contacts listed in order form.
- Incident communication process references:
  - `docs/compliance/incident-response-plan-template.md`

## Billing
- Pilot price, billing cadence, and term stated explicitly.
- Stripe billing flows are limited to configured pilot plan(s).
- Live billing starts only after successful end-to-end billing validation.

## Success criteria and exit
- Define measurable pilot success metrics.
- Define end-of-pilot options: convert to commercial plan, extend pilot, or terminate.
- Include data return/deletion terms at pilot end.
